How the Files Connect (the mental model)

Think of index.html as the hub. It doesn't hold the CSS or JS anymore — it just links to them, like this:

html
<head>
  <!-- Instead of 1,000 lines of CSS inline... -->
  <link rel="stylesheet" href="css/variables.css">
  <link rel="stylesheet" href="css/layout.css">
  <link rel="stylesheet" href="css/components.css">
</head>

<body>
  <!-- Your HTML panels, tabs, etc. -->

  <!-- Instead of 200 lines of JS inline... -->
  <script src="js/data.js"></script>
  <script src="js/player.js"></script>
</body>

The browser reads index.html, sees those links, and automatically loads the other files. You don't have to do anything special — it just works.

The Step-by-Step Process to Split It

Do these one at a time, and test in your browser after each step. Never do all of them at once — that's how you lose track of what broke something.

Step 1 — Create your folder structure first. Just make the empty folders: css/ and js/ sitting next to your index.html.

Step 2 — Extract CSS variables first. Copy everything inside :root { ... } into css/variables.css. Add <link rel="stylesheet" href="css/variables.css"> to your HTML <head>. Delete those lines from your HTML. Open in browser — everything should look identical.

Step 3 — Extract animations. Copy all your @keyframes blocks into css/animations.css. Link it. Delete from HTML. Test.

Step 4 — Extract layout CSS. Everything for the command bar, hero section, and grid goes into css/layout.css. Link, delete, test.

Step 5 — Extract component CSS. Tabs, stat cells, tables, cards, badges — into css/components.css. Link, delete, test.

Step 6 — Extract JavaScript. Copy your entire <script> block into js/player.js to start. Add <script src="js/player.js"></script> before the closing </body> tag. Delete the inline script. Test that clicking tabs and loading players still works.

Step 7 — Split the JS further (optional, do this last when you're comfortable). Separate player data into js/data.js, tab logic into js/tabs.js, etc.

One Important "Gotcha" to Know

When you open your HTML file directly by double-clicking it (a file:// URL), linked CSS and JS files sometimes won't load due to browser security rules. You need a local server. The easiest way is a free VS Code extension called Live Server — you install it, right-click your index.html, and click "Open with Live Server." That's it. This is the standard tool every web developer uses for exactly this situation.

Why This Structure Matters as You Grow

Right now splitting is about manageability. But the real payoff is later:

  • You can change all colors in one place (variables.css) without touching anything else
  • You can hand someone "just the player loading logic" (player.js) without sending them 1,900 lines
  • If something breaks, you know which file to look in immediately

The golden rule: one concern per file. Styles know nothing about behavior. Data knows nothing about layout. Each file has one job.
